# Submission Checklist (Due Feb 12, 6pm PT)

## Required
- [ ] App listing — published on developers.reddit.com (`developers.reddit.com/apps/{app-name}`)
- [ ] Demo post — public post running the game in your test subreddit
- [ ] Subreddit access — subreddit is public OR has dr-admin-approve installed
- [ ] Text description — explain features/functionality on the Devpost submission form
- [ ] README.md — detailed README in root directory describing the game and how to play
- [ ] Devpost submission — fill out all required fields at redditdailygames2026.devpost.com

## Strongly Recommended
- [ ] Demo video — under 1 minute, uploaded to YouTube/Vimeo, showing game running on device
- [ ] Mobile testing — verify game works well on mobile (eligible for Best Mobile Game Play)
- [ ] Test as a new user — experience should be self-explanatory (judges evaluate from demo post)

## Polish
- [ ] Error states — no broken states if data fails to load or unexpected user actions
- [ ] Loading experience — smooth skeleton/loading states
- [ ] Daily content working — verify today's puzzle loads correctly via bundled data
- [ ] Comments/sharing — share/comment flows work (games that drive discussion score higher)

## Optional Bonus
- [ ] Public repo link on Devpost
- [ ] Developer feedback survey (shot at $200 Feedback Award)
- [ ] Engage on Discord/r/devvit (Devvit Helper prize eligibility)

## Judging Criteria (equally weighted)
- Delightful UX
- Polish
- Reddit-y (community-minded)
- Recurring Content
